Data and API Sources

This page gives an overview of the three APIs and data sources used in our project.

SMARD

SMARD provides the electricity market data used in this project. We use SMARD data from 2022 to 2025 for electricity consumption, electricity generation, electricity prices, and regional values for Germany’s TSO control areas.

Main use in the project: analysis of national electricity demand, generation mix, regional TSO consumption and generation, and price developments.

API / Source link: https://www.smard.de/

Open-Meteo

Open-Meteo provides the weather data used in the project. We use daily weather variables such as maximum temperature, minimum temperature, wind speed, and precipitation in order to analyze how weather conditions are related to electricity consumption and renewable generation.

Main use in the project: seasonal consumption analysis and the weather-based explanation of wind and photovoltaic generation patterns.

API / Source link: https://open-meteo.com/

OpenHolidays API

The OpenHolidays API provides the holiday information used in this project. We use public holiday data and combine it with school holiday information in order to compare electricity consumption on regular working days, Sundays, and holiday-related periods.

Main use in the project: identification of public holidays and holiday periods for the analysis of electricity demand differences in Germany.

API / Source link: https://www.openholidaysapi.org/

How the Data Was Combined

We merged the electricity data, weather data, and holiday information on a daily level. This makes it possible to compare demand, generation, prices, and external factors such as temperature, wind conditions, and holidays within one combined dataset.
